Introduction

Functions are one of the fundamental building blocks in JavaScript. A function is a reusable set of statements that performs a task or calculates a value.

Functions help with:

  • Code reusability
  • Modularity
  • Abstraction
  • Organization

In JavaScript, functions are first-class objects, meaning they can be:

  • Assigned to variables
  • Passed as arguments to other functions
  • Returned from other functions
Function Declaration

A function declaration defines a named function using the function keyword:

function greet(name) {
    return `Hello, ${name}!`;
}

Key characteristics:

  • Hoisted to the top of their scope
  • Can be called before they're defined
  • Must have a name
Function Expression

A function expression defines a function as part of a larger expression, typically by assigning it to a variable:

const greet = function(name) {
    return `Hello, ${name}!`;
};

Key differences from declarations:

  • Not hoisted
  • Can be anonymous or named
  • Created when execution reaches them
Arrow Functions

Arrow functions provide a concise syntax for writing functions, introduced in ES6:

const greet = (name) => `Hello, ${name}!`;

Key features:

  • Shorter syntax
  • No own this binding (inherits from parent scope)
  • Can't be used as constructors
  • No arguments object

For multi-line functions, use curly braces:

const sum = (a, b) => {
    const result = a + b;
    return result;
};
Parameters vs Arguments

Parameters are variables listed in the function's definition. Arguments are the actual values passed to the function.

function multiply(a, b) {  // a and b are parameters
    return a * b;
}
multiply(3, 4);  // 3 and 4 are arguments

JavaScript is flexible with arguments:

  • You can pass fewer arguments than parameters (missing parameters get undefined)
  • You can pass more arguments than parameters (excess arguments can be accessed via arguments object)
Return Values

Functions can return values using the return statement:

function square(x) {
    return x * x;
}

If no return is specified, the function returns undefined.

The return statement immediately exits the function:

function checkAge(age) {
    if (age < 0) return "Invalid age";
    if (age < 18) return "Minor";
    return "Adult";
}
Scope and Closures

Functions create their own scope. Variables defined inside a function are not accessible outside it.

A closure is a function that remembers its outer variables and can access them:

function createCounter() {
    let count = 0;
    return function() {
        return ++count;
    };
}
const counter = createCounter();
counter(); // 1
counter(); // 2

The inner function maintains access to count even after createCounter has finished executing.

Higher-Order Functions

Higher-order functions are functions that operate on other functions by taking them as arguments or returning them.

Example of a function that takes another function as an argument:

function operate(a, b, operation) {
    return operation(a, b);
}
const result = operate(5, 3, (x, y) => x * y); // 15

Example of a function that returns another function:

function multiplier(factor) {
    return function(x) {
        return x * factor;
    };
}
const double = multiplier(2);
double(5); // 10
IIFEs

Immediately Invoked Function Expressions (IIFEs) are functions that run as soon as they're defined:

(function() {
    console.log("This runs immediately");
})();

Common uses:

  • Creating private scope
  • Avoiding polluting the global namespace
  • Module pattern implementation

Modern JavaScript often uses block scope (let/const) and modules instead of IIFEs.
